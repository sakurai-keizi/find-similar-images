# find_similar_images

JPEGファイルが大量に入ったフォルダを走査し、リサイズ・トリミング・再圧縮などで加工された「おそらく同じ画像」をグループ化して別フォルダに移動するツールです。

単純なファイルハッシュではなく **知覚的ハッシュ（pHash / dHash）** を使うため、画像の見た目が同じであれば検出できます。さらに **YOLOv8-pose による姿勢推定** も併用するため、構図が似ていても人物のポーズが異なる画像は別グループに分けられます。姿勢ベクトルは胴体長で正規化されるので、リサイズやトリミングがあっても比較できます。

## 必要なもの

- [uv](https://docs.astral.sh/uv/) （依存ライブラリは初回実行時に自動インストールされます）
- Python 3.9 以上

## 使い方

```
uv run find_similar_images.py <フォルダパス> [オプション]
```

### 基本的な流れ

**1. まず `--dry-run` で検出結果を確認する**

```bash
uv run find_similar_images.py /path/to/photos --dry-run
```

ファイルは移動せず、どのグループに何枚が分類されるかだけ表示します。
閾値を調整しながら繰り返し実行して、検出精度を確認してください。

**2. 問題なければ実際に移動する**

```bash
uv run find_similar_images.py /path/to/photos
```

入力フォルダ内に `similar_groups/` が作られ、グループごとのサブフォルダに移動されます。

```
/path/to/photos/
├── photo_001.jpg          ← 単独画像（移動しない）
├── photo_002.jpg          ← 単独画像（移動しない）
└── similar_groups/
    ├── group_0001/
    │   ├── photo_003.jpg
    │   └── photo_003_small.jpg
    └── group_0002/
        ├── photo_004.jpg
        └── photo_004_resized.jpg
```

グループに属さない画像（類似するものがない画像）は元の場所のまま残ります。

## オプション

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `--threshold N` | `10` | ハッシュ類似判定の閾値（ハミング距離、0〜64） |
| `--pose-threshold X` | `0.15` | 姿勢類似度の閾値（胴体長で正規化したL2距離） |
| `--output-dir NAME` | `similar_groups` | 出力ディレクトリ名（入力フォルダ内に作成） |
| `--dry-run` | — | 移動せずに検出結果だけ表示する |

### 閾値（`--threshold`）の目安

| 値 | 検出できるもの |
|----|--------------|
| 0〜5 | ほぼ同一ファイル（JPEG再圧縮程度の差） |
| **6〜10**（デフォルト） | リサイズ・画質変換された同じ画像 |
| 11〜20 | 軽度のトリミングや色調補正を含む類似画像 |
| 21〜30 | 重度のトリミングや加工も含む（誤検知が増える） |

閾値を上げるほど多く検出できますが、無関係な画像が同じグループに入る可能性も上がります。
まず `--dry-run` で結果を見ながら調整することを推奨します。

### 姿勢閾値（`--pose-threshold`）の目安

姿勢特徴は **両腰の中点を原点・肩-腰の胴体長でスケール正規化** された (x, y) 座標の集合です。距離は胴体長を基準とした単位なので、画像のサイズや切り出し範囲に依存しません。

| 値 | 検出できるもの |
|----|--------------|
| 0.10 以下 | ほぼ同じ姿勢のみ |
| **0.15**（デフォルト） | ほぼ同じ姿勢のみ許容 |
| 0.20〜0.25 | 多少の手足の角度差を許容 |
| 0.30 以上 | 姿勢の違いに寛容（誤検知が増える） |

**人物が検出できない画像（風景写真、顔アップ、胴体が隠れている等）はハッシュのみで判定されます**（フォールバック）。

### 例

```bash
# リサイズ画像のみ検出（厳密）
uv run find_similar_images.py ~/Pictures --threshold 5

# トリミング画像も含めて検出
uv run find_similar_images.py ~/Pictures --threshold 18 --dry-run

# 姿勢の違いに寛容（手足の角度違いまでまとめる）
uv run find_similar_images.py ~/Pictures --pose-threshold 0.25

# 出力先フォルダ名を変更
uv run find_similar_images.py ~/Pictures --output-dir duplicates
```

## 仕組み

1. 指定フォルダ直下の `.jpg` / `.jpeg` ファイルを列挙
2. 各画像から **pHash**（知覚的ハッシュ）と **dHash**（差分ハッシュ）を計算
3. **YOLOv8-pose** で人物の17キーポイントを検出し、両腰の中点を原点・胴体長で正規化した姿勢ベクトルを抽出
4. すべての画像ペアを比較し、以下を満たすものを「類似」と判定:
   - いずれかのハッシュのハミング距離が `--threshold` 以下、**かつ**
   - 両画像で人物が検出できた場合は姿勢ベクトル間のL2距離が `--pose-threshold` 以下（人物未検出ならハッシュのみで判定）
5. **Union-Find** で推移的にグループ化（A≈B かつ B≈C なら A・B・C が同じグループ）
6. 2枚以上のグループを出力フォルダに移動

## 注意事項

- **元に戻せません。** 移動は `shutil.move` で行われるため、事前に `--dry-run` で確認することを強く推奨します。
- サブフォルダは再帰的にスキャンしません。直下の JPEG のみが対象です。
- すでに `similar_groups/` 内にあるファイルは再処理されません（2回目実行時の重複移動を防ぎます）。
- 初回実行時に YOLOv8-pose のモデルファイル `yolov8n-pose.pt`（約6MB）がカレントディレクトリに自動ダウンロードされます。
- 姿勢推定は `ultralytics`（torch を含む）を使用するため、初回 import に数秒〜10秒程度かかります。
- YOLOv8-pose は画像内で最も信頼度の高い1人のみ使用します。集合写真ではメインの被写体だけが姿勢比較の対象になります。
